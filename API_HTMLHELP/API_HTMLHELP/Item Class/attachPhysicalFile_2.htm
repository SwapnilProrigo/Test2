Item Class

attachPhysicalFile(String filepath String vaultID)

Attach (checkin) physical file to the item to specified vault. Calling the method means that the physical will be streamed to the server when apply() is called on the item.

JavaScript:

var vault = this.newItem("Vault", "get");
vault.setProperty("name", "Alternate");
vault = vault.apply();
var vaultID = vault.getID();

var itm = this.newItem("File", "get");
itm.setProperty("filename", "Survey.xml");
itm = itm.apply();
//item needs to be locked to attach file
itm = itm.lockItem();
itm.attachPhysicalFile("c:\\Survey.xml", vaultID);
//now save the changes to File Item which will also unlock item
itm.setAction("update");
itm = itm.apply();
return itm;

Notes:

This method is deprecated in Version 11 and above - see setFileProperty to upload a physical file.

The method should only be called on client side and for items of type 'File'. Filepath can be absolute or relative path to a physical file. Relative path information is interpreted as relative to the current working directory (i.e. location of the client executable). It's possible to attach to the item a physical file with a name different than the value of item's property filename - in this case the file stored in vault will still have name and type equal to values of filename and file_type properties correspondingly. Note, that if physical file is moved or deleted between calls to this method and Item.apply(), the operation will fail.
