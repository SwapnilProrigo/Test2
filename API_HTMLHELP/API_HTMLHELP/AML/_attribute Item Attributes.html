AML Attributes

Item Attributes

Item attributes are available when adding, getting, or changing/deleting Items with AML actions.

Get Item Attributes

The following item attributes are supported when retrieving an Item(s) from the server using the get action.

id

The GUID of the item to be returned,

idlist="1D1BA69FE8CB4DD58108623740807B19"

idlist

The GUID list of the items to be returned,

idlist="1D1BA69FE8CB4DD58108623740807B19,10EEE1F2F00543AF84673D07650BDC01"

select

A comma delimited list of property names (column names) to return with each Item.

select="item_number,name"

where

A database SQL where clause to locate one or more Items. Each property name represents a column name in the database.

where="[part].is_released='1'"

orderBy

A comma delimited list of property names (column names) to order the results. Used as the ORDER BY clause in the SQL statement.

orderBy="last_name"

page

The page number for the results set.

page="1"

pagesize

The page size for the results set.

pagesize="25"

maxRecords

The maximum number of Items to be returned from the database.

maxRecords="25"

levels

The Item configuration depth to be returned. This should be used with caution as performance can be affected due to the lack of specifity in the query. Use the nested Relationships style of defining queries for faster retrievals.

levels="5"

serverEvents

If 0 then disable the server events improving performance. Default is 1.

serverEvents="0"

isCriteria

If 0 then include the nested structure for the Item configuration in the response but don't use it as search criteria. Default is 1, which will use the nested structure in the request as search criteria.

isCriteria="0"

related_expand

If 0 then do not expand the related_id Property for the relationship Items to include the related Item. Only return ID.

related_expand="0"
 

language

A comma-delimited list of language codes, or "*" to return all languages. Multilingual property

language="en,fr"

queryType

Return versioned Items based on Latest, Effective, or Released date.  Specify queryDate below.

queryType="Released"

queryDate

Used with queryType above.  Format is yyyy-mm-ddThh:mm:ss

queryDate="2013-01-02T12:00:00"

Add/Create Item Attribute

The following item attribute is available when creating or adding an Item,

id

The GUID to be set on the new Item. (Must be unique.)

id="1D1BA69FE8CB4DD58108623740807B19"

Edit/Merge/Update/Delete/Purge/Lock/Unlock Item Attributes

The following item attributes are supported when changing or removing an Item(s) from the server.

id

The GUID of the item to be returned,

id="1D1BA69FE8CB4DD58108623740807B19"

idlist

The GUID list of the items to be effected,

idlist="1D1BA69FE8CB4DD58108623740807B19,10EEE1F2F00543AF84673D07650BDC01"

where

A database SQL where clause to locate one or more Items. Each property name represents a column name in the database.

where="[part].is_released='1'"

Promote Item Attributes

The following item attributes are supported when changing or removing an Item(s) from the server.

id

The GUID of the item to be effected,

id="1D1BA69FE8CB4DD58108623740807B19"

idlist

The GUID list of the items to be effected,

idlist="1D1BA69FE8CB4DD58108623740807B19,10EEE1F2F00543AF84673D07650BDC01"

where

A database SQL where clause to locate one or more Items.

where="id like '%1D'"
